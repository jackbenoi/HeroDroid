Google Play App Store [CMS]

This page will guide you in the installation of our script. Please check all the requirements and purchase the script on codecanyon only.

Configuration and Uploading Files

  1. Please check your server requirements again.

    Server Requirements
    • PHP >= 5.5.9
    • OpenSSL PHP Extension
    • PDO PHP Extension
    • Mbstring PHP Extension
    • Tokenizer PHP Extension

  2. Unzip GooglePlayAppStore[CMS] Script after downloading it from codecanyon.

    After unzipping our script you will find, documentation folders and GooglePlayStoreApp[CMS] folder. Open up GooglePlayStoreApp[CMS] folder and edit .local.env file for your configuration. Please see the image below.

    Once you open up .local.env file, with your favorite text editor. please make sure to edit the lines of codes that was highlighted in the image, to used fully the funcitonality of the script.

    This are the basic thing that you need to edit.

    Website Link

    • APP_URL

    *Database Configurations

    • DB_DATABASE
    • DB_USERNAME
    • DB_PASSWORD

    Mail Settings

    • MAIL_DRIVER
    • MAIL_HOST
    • MAIL_PORT
    • MAIL_USERNAME
    • MAIL_PASSWORD
    • MAIL_ENCRYPTION

Storage Folder

  1. Make sure storage folder is writtable so you will not have problem in the installation of our script.

  2. If you have access to your ssh server, then do this.

    • Login in your server ssh terminal and go to your google play app store script.
    • In your terminal, run this code.. chmod -R 777 storage
    • It will run recursively and make all the folder inside becomes writable.

  3. If you cant access your terminal command you can used your filezilla ftp or cpanel to change the permission of the storage folder.

Social Login

Facebook

  1. Create Facebook App
  2. First, visit this link, Developers Facebook,
  3. After login, click the create app button and a new modal will popup.
  4. Fill up all the details, and click Create App ID. After that you will be redirected to the main dashboard. See image below.
  5. Note your App ID and App Secret.
  6. Then go to, Settings, and fill up App Domains and create new platform for your website.
  7. After clicking the + Add Platform choose the Website.

  8. Once your done, hit the button "Save Changes".

  9. Get your App ID and App Secret and paste it in our config/oauth-5-laravel.php

Google+

  1. Visit this link, console developers google, then login your gmail account or create new one.
  2. Click the create new app, and then fill up the details and click the create button.
  3. Go to library menu in the sidebar, and click Google+ API, see the image below.
  4. Then click the enable button.
  5. Click credentials menu in the sidebar and select the tab, OAuth Consent Screen and put your Product Name.
  6. Then, in the first tab Credentials, select the OuAth Client ID. see the image below.
  7. Fill up all the necessary details for your callback.. please see the instruction inside the image..

  8. Once you're done you can get already your client id and client secret.

  9. Get your client id and client secret and paste it in our config/oauth-5-laravel.php

Twitter

  1. First thing to do, visit twitter app for developers create twitter account or login your existing account.
  2. Once you login, click the create new app button. and you will be redirected to the creation page.. see image below.
  3. Fill up all the necessary details, the click the Create your Twitter Application button.
  4. Get your Consumer Key and Consumer Secret

ReCaptcha Code

  1. First visit Google reCaptcha site https://www.google.com/recaptcha/intro/android.html click "Get Recaptcha" button at the top right.

  2. It will redirect you to login using your gmail account, after logging in, you can start generating your site key and secret key.

  3. After clicking the register button, it will redirect you to the next page with the generated site and secret key, copy that and save it in the admin general settings page.

Final Step

  1. After editing all the necessary files, you can start browsing your new google play store app cms script.
  2. Try to navigating it in your browser and everything will work nicely.
